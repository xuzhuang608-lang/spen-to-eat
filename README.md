# 饭点转转

饭点转转是一款微信小程序，用城市特色美食转盘帮用户解决“今天吃什么”。首版按个人主体上线思路设计：不展示具体店铺，不接交易、不做团购、不做商家入驻，先验证用户是否愿意用“转盘 + 本轮菜单 + 饭局提议”减少饭点选择困难。

## 当前状态

- 已实现：首页、城市选择、定位、转盘、结果页、搜索、收藏历史、反馈、隐私说明、饭局提议。
- 数据已从大文件压缩为省份数据包，当前 `miniprogram` 约 1.31MB。
- 城市菜品优先展示本地菜；本地数据不足时，用省内共享菜和全国通用菜兜底。
- 转盘支持换一批、自定义加一道、替换本轮菜单里的某一道菜。
- 饭局提议支持单菜和多菜分享，不依赖云数据库，不做全群实时统计。
- 已接入高德 WebService 逆地理定位，用户也可以拒绝定位并手动选择城市。

## 核心功能

- 城市选择：按省份展开城市，也支持直接搜索城市。
- 定位切城：调用微信定位和高德逆地理接口识别当前城市，定位失败可手动选择。
- 美食转盘：按城市、餐段、分类、口味、场景、避雷条件筛选备选菜。
- 本轮菜单：展示完整菜名，支持查看详情，也支持自定义替换。
- 结果页：展示菜品文案、本地特色指数、相关菜、收藏、分享和反馈入口。
- 搜索：按城市、菜名、分类、口味和标签检索。
- 收藏历史：基于本地存储。
- 饭局提议：支持单菜“喊朋友看看”和多菜“组个饭局”，分享链接可带出同一组菜。

## 数据方案

首版采用低成本本地数据方案：

- 小程序包内只保留精简字段。
- 全国数据按省份拆分在 `miniprogram/data/provinces/`。
- 每条菜品使用短字段和枚举编码，详情文案由服务层模板生成。
- 原始完整数据备份在 `project-prep/data-backups/`。
- `miniprogram/services/dish.js` 会做菜名清洗、同城去重、共享菜过滤和全国通用菜兜底。

当前不依赖云数据库，因此开发和试用成本较低。真实多人实时统计、远程后台编辑、跨设备收藏同步等能力需要后续接入云数据库、云函数或自建后端。

## 地图配置

定位功能使用高德 WebService 逆地理接口：

- 请求域名：`https://restapi.amap.com`
- 本地私有配置：`miniprogram/config/map.js`
- 示例配置：`miniprogram/config/map.example.js`

`miniprogram/config/map.js` 已加入 `.gitignore`，不要提交真实 Key。

## 运行方式

1. 使用微信开发者工具打开项目根目录。
2. 小程序根目录为 `miniprogram/`。
3. 当前 AppID 已配置为 `wxecb26692be20801a`。
4. 在微信公众平台配置 request 合法域名：`https://restapi.amap.com`。
5. 在开发者工具中编译、预览、真机调试。

## 质检脚本

运行数据质检：

```bash
node scripts/audit_dish_quality.js
```

当前质检项包括：

- 黑名单菜名是否仍被加载。
- 同城同名菜是否重复。
- 是否存在低于 8 道可用菜的城市。

## 上线注意

- `project.private.config.json` 是本地私有配置，不提交仓库。
- `miniprogram/config/map.js` 包含地图 Key，不提交仓库。
- 首版个人主体上线应继续避免具体店铺、交易导流、商家推广、野味或高风险食材内容。
- 提交审核前建议再做一次真机全流程测试：定位、切城、搜索、转盘、结果页、收藏、饭局提议、分享。
