# 饭点转转

饭点转转是一款微信小程序，用城市特色美食转盘帮用户决定“今天吃什么”。首版定位是个人主体可上线的小程序：不展示具体店铺，不接交易、不做团购、不做商家入驻，先验证用户是否愿意用“转盘 + 本轮菜单 + 轻量投票”解决饭点选择困难。

## 当前状态

- 小程序端已实现：首页、城市选择、转盘、结果页、搜索、收藏历史、反馈、隐私说明、投票基础页面。
- 菜品数据已从完整大文件压缩为省份数据包，本地小程序体积约 1.3MB。
- 每个城市保留约 25 条候选，早餐不足时会优先用省内共享菜补齐，再用全国通用菜兜底。
- 转盘支持“换一批”和“加一道”：用户可以选择替换本轮某个菜，自选菜只参与当前转盘，不进入正式菜品库，也不跳详情。
- 当前群投票以本地存储流程为主，真实多人同步投票后续需要云数据库或自建后端。

## 核心功能

- 城市选择：按省份展开城市，也支持搜索城市。
- 美食转盘：按城市、餐段、分类、口味、场景、避雷条件筛选候选菜。
- 本轮菜单：展示完整菜名，支持点击查看详情。
- 加一道：用户可自定义替换本轮某个菜，自选菜不进入详情页。
- 结果页：展示菜品文案、本地特色指数、相关菜、收藏、分享和反馈入口。
- 搜索：按城市、菜名、分类、口味和标签检索。
- 收藏历史：基于本地存储。
- 投票：当前为轻量本地流程，后续可升级云端同步。

## 数据方案

首版采用低成本本地数据方案：

- 小程序包内只保留精简字段。
- 全国数据按省份拆分在 `miniprogram/data/provinces/`。
- 每条菜品使用短字段和枚举编码，详情文案由服务层模板生成。
- 原始完整数据备份在 `project-prep/data-backups/`。
- 明显不适合上线的内容需要持续人工复核。

当前不依赖云数据库，因此开发和试用成本较低。真实多人群投票、远程后台编辑、跨设备收藏同步等能力需要后续接入云数据库、云函数或自建后端。

## 运行方式

1. 使用微信开发者工具打开项目根目录。
2. 小程序根目录为 `miniprogram/`。
3. 当前 `appid` 为 `touristappid`，正式上传前需要替换为真实小程序 AppID。
4. 在开发者工具中编译、预览、真机调试。

## 注意事项

- `project.private.config.json` 是本地私有配置，不提交仓库。
- `project-prep/` 包含较大的调研和候选数据文件，方便追溯数据来源。
- `admin/` 目前是后台原型，含演示登录逻辑，不可直接作为线上安全后台使用。
- 首版个人主体上线应继续避免具体店铺、交易导流、商家推广、野味/高风险食材等内容。
- 上线审核前建议再做一次敏感菜名和不适合内容扫描。
