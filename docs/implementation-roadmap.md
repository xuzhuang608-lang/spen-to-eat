# 饭点转转实施路线

## 阶段 1：工程骨架与本地原型

目标：不用后端，先跑通小程序端完整主流程。

- 创建原生小程序工程
- 使用广东城市示例数据
- 完成首页、城市选择、筛选、转盘、结果页
- 完成收藏、历史、搜索、投票基础页面
- 使用本地存储模拟用户数据

验收标准：

- 微信开发者工具能导入项目
- 用户可从首页进入转盘并得到结果
- 结果可收藏、可分享、可加入投票

## 阶段 2：正式内容与后台

目标：让内容能被后台批量维护。

- 搭建后台登录
- 城市、美食、标签 CRUD
- CSV/Excel 批量导入
- 批量上下架
- 数据看板基础指标

验收标准：

- 管理员可导入首批 200 条美食
- 小程序端可从云数据库读取数据
- 可查看转盘、收藏、投票、反馈统计

## 阶段 3：云开发接入

目标：把本地 mock 替换为云开发。

- 云数据库集合设计
- 云函数接口
- openid 静默识别
- 收藏、历史、投票、反馈落库
- 隐私指引完善

验收标准：

- 不强制头像昵称授权
- 用户收藏与投票可跨会话保留
- 管理后台与小程序使用同一套数据

## 阶段 4：审核与灰度

目标：准备个人主体审核材料并灰度验证。

- 小程序简介与审核备注
- 隐私政策与定位授权说明
- 去除交易、店铺、广告、团购相关表达
- 准备测试账号和审核路径
- 小范围灰度 50-100 人

验收标准：

- 能提交微信审核
- 灰度期间收集转盘次数、分享、收藏、反馈
